TeraHAC (Hierarchical Agglomerative Clustering) Explained

TeraHAC is like building a family tree for graph nodes.

It starts with each node as its own "family" and gradually merges the closest families together, creating a hierarchy that shows how clusters form at different scales. Think of it as discovering how communities naturally group together!

Key Concept:

Hierarchical Structure: TeraHAC builds a complete hierarchy (dendrogram) showing how clusters merge at different distance levels. This provides insights into the natural grouping structure at multiple scales.

Linkage Criteria: Different linkage methods define how to measure distance between clusters - single (nearest neighbors), complete (farthest neighbors), average (all pairs), or Ward (minimize variance).

Step 1: Initialize - Start with each node as its own cluster
Step 2: Calculate - Compute distances between all cluster pairs
Step 3: Merge - Combine the two closest clusters
Step 4: Update - Recalculate distances involving the new cluster
Step 5: Repeat - Continue until reaching desired number of clusters
