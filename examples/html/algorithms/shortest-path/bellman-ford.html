Bellman-Ford Algorithm Explained

Bellman-Ford finds shortest paths even with negative edge weights.

Unlike Dijkstra's algorithm, it can handle negative weights and detect negative cycles.

The key idea: repeatedly "relax" edges to find better paths.

How Bellman-Ford Works:

What does "relaxing an edge" mean?

Relaxing an edge means asking: "Can I find a shorter path?"

For each edge from node U to node V with weight W:

  • Current shortest distance to V: distance[V]
  • New potential distance via U: distance[U] + W
  • If the new path is shorter, update it!

Example: If distance to A is 4, and edge A→B has weight 3, then distance to B via A would be 4+3=7. If current distance to B is 10, we "relax" and update it to 7.

The Algorithm Steps:

  • Step 1: Set source distance = 0, all others = ∞
  • Step 2: For V-1 iterations, relax ALL edges
  • Step 3: Try relaxing once more - if any distance improves, there's a negative cycle!

Why V-1 iterations? In the worst case, the shortest path touches every node, so it has V-1 edges. Each iteration can extend our "shortest path frontier" by one more edge.
